# ADR-005: データソース連携のスコープとフェーズ

- 日付: 2026-03-31
- ステータス: 承認

## Context (背景)

Storywright はユーザーストーリーを外部データソース（Figma、Google Spreadsheet）から取り込む。それぞれ取り込める情報の性質が異なるため、連携スコープとフェーズを明確にする必要がある。

- **Figma**: 画面デザインが中心。ストーリー（ユーザーの意図・操作手順）は含まれていない
- **Google Spreadsheet**: テスト仕様書としてURL・テスト手順・期待結果が自然言語で記述されている

### Spreadsheet の想定フォーマット

| No. | 機能/テスト項目名 | 前提条件 | テスト手順 | 期待結果 | 結果 | 担当者 | 実施日 | 備考 |
|-----|-----------------|---------|-----------|---------|------|--------|--------|------|

### Spreadsheet → Story のマッピング

| Spreadsheet列 | → Story/Step フィールド |
|---------------|----------------------|
| No. | story.id の元 |
| 機能/テスト項目名 | story.title |
| 前提条件 | story.steps[0]（navigate等の初期ステップ） |
| テスト手順 | story.steps[]（自然言語からの変換が必要） |
| 期待結果 | story.steps[] の assert ステップ |
| 結果/担当者/実施日 | テスト実行結果（SQLite側、インポート対象外） |

## Options (選択肢)

### Spreadsheet: 自然言語 → ステップ構造の変換方式

| 方式 | 説明 |
|------|------|
| A. 手動マッピング | インポート後にGUI上でステップを編集 |
| B. AI変換 | LLMで自然言語 → ステップ構造に自動変換 |
| C. ハイブリッド | AIが変換を提案 → ユーザーが確認・修正 |

### Figma: 連携の深さ

| レベル | 内容 | 必要なもの |
|--------|------|-----------|
| L1. 参照のみ | URLリンク + スクリーンショットを story.screen に保存 | MCP or API |
| L2. デザイン情報活用 | MCP経由でコンポーネント情報取得、AI変換でステップ提案 | MCP + LLM |
| L3. Plugin連携 | デザイナーがFigma上でアノテーション → ストーリー自動生成 | Figma Plugin 開発 |

## Decision (決定)

### フェーズ分け

| フェーズ | Spreadsheet | Figma | 変換方式 |
|---------|-------------|-------|---------|
| **v1** | インポート → 手動マッピング | URLリンク + スクリーンショット参照 | A（手動） |
| **v2** | AI変換提案 → ユーザー確認・修正 | MCP経由でデザイン情報取得 + AI変換提案 | C（ハイブリッド） |
| **v3** | — | Figma Plugin でアノテーション → 自動生成 | Plugin + 自動 |

### v1 で実現すること

**Spreadsheet連携:**
- Google Sheets API でスプレッドシートを読み込み
- 列マッピングをGUI上で設定（どの列がtitle、どの列がテスト手順か）
- インポート後、GUI上でステップの action/target/value を手動で編集
- 自然言語のテスト手順は description として保持し、構造化は手動

**Figma連携:**
- ストーリー作成時に Figma URL を貼り付けて画面を紐付け
- MCP or API 経由でスクリーンショットを取得・保存
- ストーリーの視覚的な参照として利用（テスト生成には使わない）

### v2 以降の拡張

**AI変換（v2）:**
- Spreadsheet のテスト手順（自然言語）を LLM でステップ構造に変換
- 変換結果をユーザーに提示し、確認・修正してもらう
- Figma のデザイン情報（コンポーネント名、レイアウト）を target のヒントに活用

**Figma Plugin（v3）:**
- デザイナーが Figma 上で操作対象にアノテーションを付与
- Plugin からエクスポート → Storywright がインポート
- アノテーション情報があればストーリーの自動生成が現実的になる
- ただし Plugin 開発は別プロジェクト級のスコープ

## Consequences (影響)

### メリット
- v1 だけで「Spreadsheetからインポート → テスト実行」の価値が出せる
- Figma連携を段階的に深められ、初期の実装スコープを抑制できる
- 手動マッピングから始めることで、AI変換のための学習データ・パターンも蓄積できる

### デメリット
- v1 では自然言語 → ステップの変換が手動で、ユーザーの作業負荷がある
- Figma Plugin（v3）は実現までの道のりが長い

### 今後の検討事項
- Google Sheets API の認証方式（OAuth? サービスアカウント?）
- Spreadsheet の列マッピングUIの設計
- v2 のAI変換に使うLLMの選定・コスト
