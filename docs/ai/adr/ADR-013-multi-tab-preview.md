# ADR-013: Preview の複数タブ対応

- 日付: 2026-04-03
- ステータス: 提案

## Context (背景)

### 前提

ADR-012 でブラウザバーと URL 統合を実現し、単一タブでのブラウザ体験は整った。

### 既知の不具合

- `target="_blank"` でタブが開いても、新タブに追従しない（現状は単一 webview のため追跡不可）。本 ADR で対応する。
しかし、以下のユースケースにはまだ対応できない:

1. **`target="_blank"` リンク**: 決済フローなどで別タブが開くが、追跡できない
2. **複数ページの並行操作**: 管理画面とユーザー画面を同時に開いて録画したい
3. **タブ間遷移の録画**: タブを切り替えながら操作するシナリオを記録したい

### あるべき体験

Chrome のタブと同じ感覚で複数ページを操作でき、録画もタブをまたいで継続される。

## Options (選択肢)

### A. タブバーを Preview に追加（フル実装）

- タブバー UI（追加・閉じる・切り替え）
- 各タブに独立した webview
- `target="_blank"` で自動新タブ
- 録画時の複数タブ追跡

**メリット**: Chrome に近い完全な体験
**デメリット**: webview 管理が複雑、メモリ使用増加

### B. タブなし、ポップアップは同一 webview で開く

- `target="_blank"` をインターセプトして同一 webview で遷移
- 戻るボタンで元に戻れる

**メリット**: 実装がシンプル
**デメリット**: 複数ページ並行操作ができない、元のページ状態が失われる

## Decision (決定)

**A. タブバーをフル実装する。**

理由:
- 決済フローなど別タブが開くシナリオは現実に多い
- 録画の対象範囲を広げるにはタブ対応が不可欠
- ADR-012 のブラウザバーの上にタブバーを載せる自然な拡張

### 変更内容

#### 1. レイアウト

```
┌─ Preview ──────────────────────────────────────────┐
│ Tab Bar                                             │
│ ┌──────┐ ┌──────┐ ┌───┐                            │
│ │ Tab 1│ │ Tab 2│ │ + │                            │
│ └──────┘ └──────┘ └───┘                            │
│ Browser Bar (ADR-012)                               │
│ ← → ↻ │ https://example.com/login           │ ▼ │ │
│                                                     │
│ ┌─ webview ───────────────────────────────────────┐ │
│ │              (アクティブタブの内容)               │ │
│ └─────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

#### 2. タブバーの機能

- 各タブにタイトル（ページの `<title>`）と ✕ ボタン
- `+` ボタンで空白タブを追加
- `target="_blank"` リンクで自動的に新タブが開く（`allowpopups` + `new-window` イベント）
- タブ切り替えで webview を切り替え（ブラウザバーの URL も連動）
- 最後のタブを閉じたら空白タブを表示

#### 3. タブの状態管理

```typescript
type Tab = {
  id: string;
  title: string;
  url: string;
  canGoBack: boolean;
  canGoForward: boolean;
};

// 状態
const [tabs, setTabs] = useState<Tab[]>([initialTab]);
const [activeTabId, setActiveTabId] = useState(initialTab.id);
```

- 各タブが独立した webview を持つ
- 非アクティブタブの webview は `display: none` で保持（状態を維持）
- ブラウザバーはアクティブタブの webview を操作

#### 4. 録画との連携

- 録画中（REC）は全タブの webContents にレコーダーをアタッチする
- 新タブが開いた際、自動的にレコーダースクリプトを注入
- タブ切り替え時の操作もステップとして記録可能
- `new-window` イベントで新タブの webContents を追跡

### 影響するファイル

| ファイル | 変更内容 |
|---------|---------|
| `src/components/PreviewPanel.tsx` | タブバー追加、複数 webview 管理 |
| `src/App.tsx` | タブ状態管理 |
| `src/App.css` | タブバーのスタイル |
| `electron/main.ts` | 複数 webContents への録画アタッチ |
| `electron/preload.ts` | 新タブ検出の IPC 追加（必要に応じて） |

## Consequences (影響)

### メリット
- 別タブが開く決済フローなどの録画が可能になる
- 複数ページを並行操作できる
- Chrome に近い完全なブラウザ体験

### デメリット
- 複数 webview の管理でメモリ使用量が増加
- 録画ロジックが複数タブ分に複雑化
- 非アクティブタブの webview を保持するためリソース消費が増える

### 影響を受ける既存 ADR

| ADR | 影響 |
|-----|------|
| ADR-012 | 拡張 — ブラウザバーの上にタブバーを追加 |
| ADR-009 | 拡張 — 録画の複数タブ対応 |
