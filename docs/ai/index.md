# docs/ai 運用ガイド

## 目的
AI設計・仕様・ADR（設計判断記録）を体系的に管理し、プロジェクトの意思決定や仕様を誰でも追いやすくするためのディレクトリです。

## ディレクトリ構成

- docs/ai/index.md …… この運用ガイド・テンプレート例
- docs/ai/product-overview.md …… プロダクト概要
- docs/ai/adr/ …… ADR（Architecture Decision Record）を格納
- docs/ai/（その他仕様書や設計資料）

## 運用ルール

- 設計判断や重要な意思決定はADRとして docs/ai/adr/ に記録
- 機能仕様や画面仕様は docs/ai/ 配下に spec-xxxx.md などで管理
- テンプレートはこのindex.mdにも記載（下記参照）
- skills化や自動生成も推奨

---

## ADRテンプレート例

```
# ADR-XXX: タイトル

- 日付: YYYY-MM-DD
- ステータス: 提案中 / 承認 / 却下

## Context (背景)
なぜこの決定が必要か、どんな課題があるかを記述。

## Decision (決定)
採用する案・選択肢とその理由を記述。

## Consequences (影響)
この決定によるメリット・デメリット、今後の影響などを記述。
```

---

## 機能仕様テンプレート例

```
# 機能仕様タイトル

## 概要
機能や画面の概要説明。

## 要件
- 要件1
- 要件2

## 画面イメージ・UI
（必要に応じて図やワイヤーフレームを挿入）

## API仕様
（必要に応じてAPIインターフェースやデータ構造を記述）

## 備考
その他注意点や補足事項。
```
