Git commit を実行します。このコマンド経由でのみ commit が許可されます。

## 手順

1. **品質チェック（自動）** — 以下を順番に実行し、すべて通過しなければ commit しない
   - `pnpm tsc --noEmit` — 型チェック
   - `pnpm build` — ビルド確認（Rust側のビルドは除く、フロントエンドのみ）
   - エラーがあればユーザーに報告し、修正を提案する。commit は中止
2. `git status` で変更状態を確認し、ユーザーに表示する
3. `git diff --staged` と `git diff` で変更内容を確認する
4. 変更内容を要約し、コミットメッセージの案を提示する
5. **ユーザーの承認を待つ** — コミットメッセージと対象ファイルを確認してもらう
6. 承認を得たら `git add` → `git commit` を実行する

## ルール

- 品質チェックが通らない限り、commit は絶対に実行しない
- ユーザーが承認するまで `git add` も `git commit` も実行しない
- .env、credentials、secrets を含むファイルは警告を出す
- コミットメッセージはユーザーが修正可能
- 引数 "$ARGUMENTS" がある場合はコミットメッセージのヒントとして使う
