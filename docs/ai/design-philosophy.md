# Storywright デザイン思想

## コアコンセプト: Warm Functional（温かい機能美）

Linear の機能美（余計なものがない、情報が整理されている）に、FigJam の親しみやすさ（操作対象が明確、遊び心、心理的安全性）を融合する。

> 仕事のツールだけど、少し楽しい。

### デザイン原則

1. **直感的に触れる** — 初見で「何をどう触ればいいか」がわかる。アイコンだけにせず、ラベルを添える。ボタンは大きめに。
2. **温かみのある余白** — 冷たいモノクロではなく、温かみのあるグレーとアクセントカラーで落ち着きと親しみを両立する。
3. **成功を祝い、失敗は冷静に** — テスト成功は嬉しく、失敗は責めずに次のアクションを示す。
4. **見えることは安心** — テスト実行中の進行状況、保存状態、エラー箇所など、システムの状態が常にユーザーに伝わる。
5. **30分で使える** — チュートリアルなしで基本操作が理解できる。非エンジニアが怖がらないUI。

## ビジュアルトーン

### 色彩

| 用途 | 方針 |
|------|------|
| 基調色 | 温かみのあるグレー（青みを抑えた暖色寄り） |
| 背景 | オフホワイト〜ライトウォームグレー |
| アクセント | 落ち着いた明るい色（ブルー or パープル系、操作可能な要素に使用） |
| 成功 | 緑（温かみのある、鮮やかすぎない） |
| 失敗 | 赤（警告的だが攻撃的でない） |
| エッジ（ストーリー） | グレー → 実行中はアクセント色 → 成功で緑 / 失敗で赤 |

### 形状

- **角丸**: 大きめ（8-12px）。角張らせない。触れる感じを出す
- **シャドウ**: 軽め。浮いている感を出すが重くしない
- **線**: ほんの少し柔らかさのあるライン（完全な直線よりカーブ）
- **ノード（画面カード）**: カード型、スクリーンショットが映える余白

### タイポグラフィ

- 本文: 読みやすいサンセリフ（Inter, Noto Sans JP など）
- 見出し: やや太めのウェイトで視覚的階層を作る
- コード・セレクタ表示: モノスペースで区別する

## マイクロインタラクション

### テスト実行

| シーン | 演出 |
|--------|------|
| ステップ実行中 | エッジ上を光の粒が流れる。進行中のステップがハイライト |
| テスト成功 | ノードが一瞬パルスして緑に光る。控えめな紙吹雪 |
| テスト失敗 | ノードが軽くシェイク（震え）+ 赤くハイライト。失敗箇所にフォーカス移動 |
| 全テスト通過 | キャンバス全体に控えめな祝福エフェクト |
| 連続成功 | ストリーク表示（「3連続成功！」） |

### 日常操作

| シーン | 演出 |
|--------|------|
| ストーリー接続 | エッジがゴムバンドのように伸びて吸着 |
| ノードのドラッグ | 軽い慣性でスムーズについてくる |
| ステップ追加 | リストにすとんと落ちるアニメーション |
| 保存完了 | ツールバーにチェックマークがふわっと表示 → フェードアウト |
| 初めてのテスト成功 | 「最初のテストが通りました！」のトースト通知 |

### 原則

- **嬉しい瞬間は祝う、失敗は冷静に伝える**
- アニメーションは 200-400ms。長すぎると邪魔、短すぎると気づかない
- すべての演出はオフにできる設定を用意する（好みに配慮）
- 操作を邪魔しない。演出中もキャンバス操作は受け付ける

## レイアウト

### 基本構成

```
┌──────────────────────────────────────────────────┐
│ Toolbar                                           │
│ [+ 画面追加] [+ ストーリー追加] [Import] [▶ Run All]│
├──────────────────────────────────────────────────┤
│                         │                         │
│                         │  Detail Panel            │
│   Canvas                │                         │
│   (React Flow)          │  選択中のストーリーの     │
│                         │  ステップ編集             │
│   [画面A] ──→ [画面B]    │  テスト結果表示           │
│                         │                         │
│                         │                         │
├──────────────────────────────────────────────────┤
│ Status Bar                                        │
│ テスト実行状態 / 最終保存時刻 / 接続中のブラウザ      │
└──────────────────────────────────────────────────┘
```

### レイアウト原則

- **Canvas が主役** — 画面の最大面積を確保する
- **Detail Panel は右側** — 選択に応じて内容が切り替わる。折りたたみ可能
- **Toolbar は最小限** — よく使う操作だけ。詳細設定はパネル内
- **Status Bar** — テスト実行中の進捗、保存状態など常に見える情報

## このドキュメントの運用

- UI に関わるコード変更は、このドキュメントのデザイン原則に従う
- 新しいコンポーネントを追加する際は、既存のビジュアルトーンとの一貫性を確認する
- デザイン判断に迷った場合は、5つのデザイン原則に立ち返る
- CSSフレームワーク・コンポーネントライブラリの選定時はこの思想との整合性を基準とする
