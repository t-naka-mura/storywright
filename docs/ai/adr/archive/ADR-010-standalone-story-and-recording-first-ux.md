# ADR-010: スタンドアロン Story と録画ファースト UX

- 日付: 2026-04-01
- ステータス: 承認

## Context (背景)

### 現状の課題

ADR-009 でインタラクティブレコーダーを実装し、Preview をアプリ内に埋め込んだことで、Preview がデフォルト表示となった。しかし、現在の録画フローには以下の問題がある:

1. **録画前にストーリー（エッジ）の事前選択が必要**: Canvas でエッジをクリック → Preview に切り替え → REC という手順が必要
2. **Preview デフォルト表示との不整合**: アプリ起動時に Preview が表示されるが、Detail Panel は「エッジ（ストーリー）をクリックするとステップが表示されます」と Canvas 前提のメッセージを表示
3. **Story が Edge に強く結合**: ADR-004 ではフラット構造の Story を定義しているが、現在の実装では Story が Canvas のエッジ（画面間の接続）に1:1で紐づいており、エッジなしに Story を作成できない

### あるべき体験

「Preview を開く → REC を押す → 操作する → Stop → テストが完成」という直感的なフローを実現したい。非エンジニアが30分で使える（デザイン原則5）ためには、Canvas の概念を理解せずとも録画からテストを作成できる必要がある。

## Options (選択肢)

### A. REC で新規スタンドアロン Story を自動生成

- REC ボタンを押すと、エッジに紐づかない「スタンドアロン Story」が自動作成される
- 録画中のステップはこの Story に追加される
- Stop 後、Story は Detail Panel のリストに表示される
- 後から Canvas 上のエッジに紐づけることも可能

**メリット**: シンプル、録画ファーストの体験を実現、既存のキャンバスフローも維持
**デメリット**: Story とエッジの紐づけが任意になり、管理が複雑になる可能性

### B. REC で Story + 画面ノードも自動生成

- navigate を検出するたびに画面ノードを自動作成
- Stop 時に Story + 画面遷移グラフが Canvas に反映される

**メリット**: 録画から Canvas が自動構築される
**デメリット**: 画面の自動判定が複雑、ノイズ的なノードが生成されるリスク

### C. Panel のメッセージ変更のみ（最小変更）

- Preview 表示時の Detail Panel メッセージを「● REC を押して録画を開始してください」に変更
- Story とエッジの結合は維持し、事前にエッジ選択を必須のままにする

**メリット**: 最小の変更量
**デメリット**: 根本的な UX 問題は解決しない、非エンジニアには依然として Canvas の理解が必要

## Decision (決定)

**A. REC で新規スタンドアロン Story を自動生成する。**

### 変更内容

#### Story の管理方式

- Story はエッジに紐づくもの（従来）と、スタンドアロン（新規）の2種類になる
- スタンドアロン Story は `stories` state に直接追加され、エッジとの紐づけは任意
- Detail Panel にストーリー一覧を表示し、エッジ選択以外からもアクセス可能にする

#### 録画フロー（新）

```
1. アプリ起動 → Preview 表示（デフォルト）
2. Base URL を入力 → サイトが Preview に表示される
3. ● REC をクリック
   → 新しいスタンドアロン Story が自動生成（タイトル: "録画 YYYY-MM-DD HH:mm"）
   → Detail Panel にステップがリアルタイム表示
4. サイトを操作する（click, type, navigate, select）
5. 必要に応じて ✓ Assert で検証を追加
6. ■ Stop で録画終了
7. Story のタイトルを編集、▶ Run でテスト実行
```

#### Detail Panel の変更

- Preview タブ表示時のデフォルトメッセージを変更:
  - Story 未選択: 「● REC を押して録画を開始するか、Canvas でストーリーを選択してください」
  - スタンドアロン Story 一覧があれば表示
- ストーリー一覧（サイドバー的な役割）の追加は将来検討

#### REC ボタンの変更

- ストーリー未選択でも REC が押せるようになる（`canRecord` の条件を緩和）
- REC 押下時に Story が未選択なら、新規スタンドアロン Story を自動生成

## Consequences (影響)

### メリット
- 「録画 → テスト」の最短フローが実現される
- 非エンジニアが Canvas を理解せずにテストを作成できる
- 既存のキャンバスベースのワークフローは維持される（破壊的変更なし）

### デメリット
- Story の管理が2系統（エッジ紐づき / スタンドアロン）になる
- スタンドアロン Story が増えた場合の一覧・検索 UI が将来必要になる

### 影響を受ける既存 ADR

| ADR | 影響 |
|-----|------|
| ADR-004 | 拡張 — Story にスタンドアロン（エッジ未紐づけ）の概念を追加 |
| ADR-006 | 軽微 — Detail Panel のデフォルト表示を変更 |
| ADR-009 | 拡張 — 録画フローの簡略化 |
