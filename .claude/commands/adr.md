ADR（Architecture Decision Record）を作成してください。

## 手順

1. docs/ai/adr/ 内の既存ADRを確認し、次の番号を決定する（例: ADR-001 が最後なら ADR-002）
2. ユーザーの入力 "$ARGUMENTS" をもとに、以下のフォーマットでADRを作成する
3. ファイル名は `ADR-{番号}-{kebab-case-タイトル}.md` とする
4. 作成先は `docs/ai/adr/` 配下

## フォーマット

```markdown
# ADR-{番号}: {タイトル}

- 日付: {今日の日付 YYYY-MM-DD}
- ステータス: 提案中

## Context (背景)
なぜこの決定が必要か、どんな課題があるかを記述。

## Options (選択肢)
検討した選択肢を列挙し、それぞれのメリット・デメリットを記述。

## Decision (決定)
採用する案とその理由を記述。
ユーザーとの合意が取れていない場合は「未決定」とし、選択肢の提示にとどめる。

## Consequences (影響)
この決定によるメリット・デメリット、今後の影響などを記述。
```

## ルール

- ステータスは「提案中」で作成する。ユーザーが承認したら「承認」に更新する
- Decisionが未確定の場合は、選択肢を提示してユーザーに判断を仰ぐ
- 作成後、docs/ai/product-overview.md の「今後検討が必要な事項」に関連があれば更新する
