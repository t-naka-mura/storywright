# ADR-007: キャンバス操作とステップ編集の仕様

- 日付: 2026-04-01
- ステータス: 廃止（ADR-015 により Canvas 廃止）

## Context (背景)

ADR-006 で React Flow によるキャンバスベースUIの採用が決まった。レイアウト骨格（Toolbar / Canvas / Detail Panel / Status Bar）は実装済みだが、各UI要素の操作仕様が未定義である。

ユーザーがキャンバス上で「画面（ノード）」と「ストーリー（エッジ）」を自由に構成し、Detail Panel でストーリーのステップ（E2Eテストの操作手順）を定義できる必要がある。

### 前提

- データモデルは ADR-004 のフラット構造（Story + Steps）
- ノード = 画面（URL + スクリーンショット参照）
- エッジ = ストーリー（画面間の遷移シナリオ）
- ステップ = ストーリー内の個々の操作（navigate / click / type / assert 等）

## Decision (決定)

### 1. ノード（画面）の操作

| 操作 | トリガー | 動作 |
|------|----------|------|
| 追加 | Toolbar「+ 画面追加」or キャンバスダブルクリック | デフォルト名で新規ノード作成、即座に名前編集モードに入る |
| 移動 | ドラッグ | 自由配置（グリッドスナップはオプション） |
| 編集 | ノードをダブルクリック | インライン編集モード（画面名・URL を直接書き換え） |
| 削除 | ノード選択 → Delete キー or コンテキストメニュー | 確認ダイアログ後に削除（接続中のエッジも連動削除） |
| 選択 | クリック | ノードをハイライト、Detail Panel に画面情報を表示 |
| 複数選択 | Shift + クリック or ドラッグ矩形選択 | 一括移動・一括削除に対応 |

#### ノード編集モードの仕様

```
┌─────────────────────┐
│ [画面名] ← input    │
│ [URL]    ← input    │
│                     │
│ [Save] [Cancel]     │
└─────────────────────┘
```

- ダブルクリックで編集モードに切り替え
- Enter で確定、Escape でキャンセル
- フォーカスが外れたら自動確定

### 2. エッジ（ストーリー）の操作

| 操作 | トリガー | 動作 |
|------|----------|------|
| 作成 | ノードのハンドルからドラッグ → 別のノードにドロップ | 新規ストーリーを作成、デフォルト名「新しいストーリー」 |
| 選択 | エッジ（線 or ラベル）をクリック | エッジをハイライト、Detail Panel にステップ一覧を表示 |
| ラベル編集 | エッジのラベルをダブルクリック | インライン編集モードでストーリー名を変更 |
| 削除 | エッジ選択 → Delete キー or コンテキストメニュー | 確認ダイアログ後に削除（ステップデータも連動削除） |
| 再接続 | エッジの端点をドラッグ | 接続先ノードを変更 |

### 3. ステップ編集（Detail Panel）

エッジ選択時に Detail Panel に表示されるステップ編集UI。

#### レイアウト

```
┌─────────────────────────────┐
│ ストーリー名: ログインフロー    │
│ ─────────────────────────── │
│ ① navigate  /login          │  ← ドラッグで並べ替え
│ ② type  #email → test@...   │
│ ③ click  .submit            │
│ ④ assert  .title → "..."    │
│                             │
│ [+ ステップ追加]              │
│ ─────────────────────────── │
│ [▶ Run] [Delete Story]      │
└─────────────────────────────┘
```

#### ステップの操作

| 操作 | トリガー | 動作 |
|------|----------|------|
| 追加 | 「+ ステップ追加」ボタン | リスト末尾に新規ステップ追加、action 選択状態で表示 |
| 編集 | ステップ行をクリック | 展開してフォーム表示（action / target / value / description） |
| 削除 | ステップ行のゴミ箱アイコン or Delete キー | 即時削除（確認なし、Undo で復元可能） |
| 並べ替え | ドラッグ＆ドロップ | order を自動再計算 |
| 複製 | コンテキストメニュー「複製」 | 直下にコピーを挿入 |

#### ステップ編集フォーム（展開時）

```
┌─────────────────────────────┐
│ Action: [navigate ▼]        │  ← プルダウン
│ Target: [/login         ]   │  ← テキスト入力
│ Value:  [              ]    │  ← テキスト入力（action により表示/非表示）
│ 説明:   [ログインページに遷移] │  ← 任意
│                [Save] [×]   │
└─────────────────────────────┘
```

- action の種別により、target / value の入力ガイドを切り替え:
  - `navigate`: value にURL、target は非表示
  - `click`: target にセレクタ、value は非表示
  - `type`: target にセレクタ、value に入力値
  - `assert`: target にセレクタ、value に期待値
  - `wait`: target にセレクタ、value に条件（hidden / visible / timeout）
  - `screenshot`: description のみ
  - `select`: target にセレクタ、value に選択値

### 4. キーボードショートカット

| ショートカット | 動作 |
|-------------|------|
| Delete / Backspace | 選択中のノード or エッジを削除 |
| Cmd + Z | Undo |
| Cmd + Shift + Z | Redo |
| Cmd + A | 全選択 |
| Escape | 選択解除 / 編集モード終了 |
| Space + ドラッグ | キャンバスのパン |

### 5. Undo / Redo

- キャンバス上のすべての操作（ノード追加/削除/移動、エッジ追加/削除、ステップ編集）を Undo / Redo 対象とする
- 履歴は最大 50 操作を保持
- Undo 対象はキャンバス状態のスナップショットではなく、操作単位（コマンドパターン）とする

## Consequences (影響)

### メリット
- キャンバス上でストーリーの構造を視覚的に組み立てられる
- ステップ編集と画面遷移が一画面で完結し、コンテキストスイッチが少ない
- Undo / Redo により安心して試行錯誤できる

### デメリット
- インライン編集やドラッグ並べ替えの実装コストが高い
- Undo / Redo のコマンドパターン実装が複雑
- キーボードショートカットがOSやブラウザと競合する可能性

### 実装の優先順位（提案）

1. **Phase 1**: ノード追加・編集・削除、エッジ接続・ラベル編集・削除
2. **Phase 2**: ステップ CRUD（追加・編集・削除・並べ替え）
3. **Phase 3**: Undo / Redo、キーボードショートカット、複数選択
